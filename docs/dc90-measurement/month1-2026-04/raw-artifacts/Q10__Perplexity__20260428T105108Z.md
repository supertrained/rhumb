Package manager for agent services/APIs

The term you’re probably looking for is APM, the Agent Package Manager: an open-source dependency manager for AI agents that treats agent configuration like software packages, with a manifest and lockfile for reproducible installs.
microsoft.github
+1

What it does

APM is meant to manage agent skills, prompts, instructions, tools, hooks, plugins, and MCP servers in one place, similar to how npm or pip manages code dependencies. It also pins exact versions so teams can share the same agent setup across environments.
microsoft.github

Why it matters

For agent services and APIs, the big idea is to package API dependencies and agent behavior the way developers already package code, so setups are repeatable and easier to audit. That makes it useful for teams building AI agents that rely on multiple services or MCP-based tools.
