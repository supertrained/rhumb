    ## Extracted response

    **Capability discovery** is the process by which an AI agent identifies, understands, and learns how to use the tools, APIs, and functions available to it within a given environment. As agents move from static scripts to autonomous systems, discovery becomes the bridge between "knowing what to do" and "knowing how to do it."

### Core Mechanisms of Discovery

1.  **Semantic Search (Vector Embeddings):**
    Agents often navigate "toolbanks" containing hundreds of functions. Instead of loading every definition into the context window (which is token-expensive), systems use RAG (Retrieval-Augmented Generation). The agent’s current goal is embedded as a vector and matched against a database of tool descriptions to retrieve only the most relevant capabilities.

2.  **Self-Describing Schemas:**
    Modern discovery relies on standardized formats like **JSON Schema** or **OpenAPI**. A tool "announces" itself with:
    *   **Name:** `get_weather_forecast`
    *   **Description:** "Fetches a 5-day forecast for a given city."
    *   **Parameters:** `city` (string), `units` (enum: metric/imperial).
    The LLM uses these descriptions to decide if the tool fits its plan.

3.  **Handoffs and Orchestration:**
    In multi-agent systems, discovery involves identifying other specialized agents. A "Manager Agent" might query a registry to find a "SQL Specialist" or "Image Generator" based on the task's requirements.

4.  **Introspection and Reflection:**
    Advanced agents can "probe" their environment. For example, an agent might run `ls` or `help()` in a terminal to discover available CLI tools, or read a `README.md` to understand a local codebase’s architecture before attempting an edit.

### Why Discovery Matters

*   **Scalability:** It allows agents to operate in environments with thousands of potential actions without overwhelming their "working memory" (context window).
*   **Dynamic Environments:** If a new API is added to a system, a discovery-enabled agent can start using it immediately without being re-trained.
*   **Interoperability:** Standardized discovery protocols (like the **Model Context Protocol - MCP**) allow different AI models to interact with a unified set of data sources and tools across different platforms.

### Emerging Standards
*   **MCP (Model Context Protocol):** An open standard that enables developers to provide a consistent way for AI agents to discover data and tools.
*   **LangChain / LangGraph:** Frameworks that provide built-in "Toolkits" and registries for agentic discovery.
*   **Semantic Kernel:** Uses "Plugins" that the kernel can search and invoke dynamically based on user intent.

### The "Description" Bottleneck
The biggest challenge in capability discovery is the **quality of natural language descriptions**. If a tool's description is vague (e.g., `process_data`), the agent may fail to discover it or use it incorrectly. Effective discovery requires "Prompt Engineering" for the tools themselves—writing descriptions that are optimized for LLM understanding.
