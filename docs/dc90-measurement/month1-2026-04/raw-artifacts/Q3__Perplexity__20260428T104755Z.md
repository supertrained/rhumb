Agent-native service directory

“Agent-native service directory” most likely refers to the Agent Directory Service (ADS), a distributed directory for discovering AI agents and agent applications by their skills and metadata. It stores agent records, uses a skill taxonomy for discovery, and supports publishing and querying agent descriptions across a network of directories.

What it does

ADS is designed to help people and systems find agents that can solve specific problems by matching on skills, attributes, and constraints. It uses a distributed architecture so directories can interconnect and stay in sync rather than relying on one central catalog.

How it works

Agent records are modeled with the Open Agentic Schema Framework, and the system uses a content-routing approach to map skills to records and locate the servers hosting them. The discovery model is taxonomy-driven, so queries can search across multiple capability dimensions instead of just a single tag.
