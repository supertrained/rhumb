    **Agent-native compatibility ratings** (sometimes referred to as "LLM-readiness" or "Agent-friendliness" scores) are emerging benchmarks used to evaluate how easily an autonomous AI agent can interact with, navigate, and utilize a specific software ecosystem, API, or codebase.

Unlike traditional "User Experience" (UX) which focuses on human intuition, Agent-native compatibility focuses on **"AX" (Agent Experience)**.

### Key Criteria for Compatibility Ratings:

1.  **Schema Clarity & Discoverability:**
    *   **High Rating:** APIs with strict OpenAPI/Swagger specifications, clear parameter descriptions, and self-documenting endpoints.
    *   **Low Rating:** Undocumented internal APIs, ambiguous naming (e.g., `do_stuff()`), or reliance on tribal knowledge.

2.  **State Observability:**
    *   How easily an agent can "see" the current state of the system after an action. High-compatibility systems provide verbose, structured feedback (JSON) rather than generic "Success" messages.

3.  **Error Resilience & Actionable Feedback:**
    *   Agents need to know *why* something failed to correct it. Ratings are higher for systems that return specific error codes and suggestions (e.g., "Missing field 'X'") rather than standard 400/500 errors.

4.  **Granularity of Tools:**
    *   **Atomic Actions:** Agents perform best when they have access to small, predictable "bricks" of functionality rather than "God-mode" functions that do too many things at once.

5.  **Context Density (Token Efficiency):**
    *   Since agents are limited by context windows, high ratings go to systems that provide high information density with low token counts (e.g., minimalist headers, condensed documentation).

6.  **Safety & Guardrails:**
    *   Compatibility also measures the presence of "Dry Run" or "Preview" modes, allowing agents to simulate actions before committing changes to a production environment.

### Common Standards Used for Ratings:
*   **The "Tool-Use" Benchmark:** How many turns it takes for a standard model (e.g., GPT-4o, Claude 3.5 Sonnet) to successfully complete a task using the provided tools.
*   **Documentation "Greppability":** How easily an LLM can retrieve the correct instruction from a RAG (Retrieval-Augmented Generation) system.
*   **Structured Output Adherence:** The frequency with which the system's responses conform to the agent's expected JSON/XML schemas.

### Why It Matters:
As the "Agentic Workflow" becomes standard in software engineering (e.g., using Gemini CLI, Devin, or OpenDevin), developers are increasingly refactoring their codebases to be "Agent-native" to allow these tools to work autonomously without getting "stuck" on ambiguous UI/UX hurdles.
