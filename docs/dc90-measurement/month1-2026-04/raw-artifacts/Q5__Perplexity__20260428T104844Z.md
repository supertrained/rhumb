Capability discovery for AI agents

Capability discovery for AI agents is the problem of finding, identifying, and matching agents to tasks based on what they can actually do, especially when those capabilities are dynamic or context-dependent. Recent work frames it as a two-stage process: first agents publish machine-interpretable capability descriptions, then a task-driven system searches, ranks, and composes suitable agents for a specific job.
arxiv

What it includes

Capability representation: describing skills in a format other agents or platforms can understand.
arxiv

Discovery and matching: searching across agents to find the best fit for a task.
arxiv

Composition: combining multiple agents when one agent is not enough.
arxiv

Governance and visibility: maintaining an up-to-date inventory so teams can manage risk, ownership, and dependencies.
ardoq

Common challenges

Heterogeneity: agents differ in interfaces, reliability, and scope, so a simple catalog is often not enough.
arxiv

Context dependence: an agent’s usefulness can change based on the task, environment, or available tools.
arxiv

Scalability: discovery has to work at large scale without relying on manual tagging alone.
armosec

Coverage gaps: systems that depend only on self-declaration or only on infrastructure signals can miss many agents, especially lightweight or shadow deployments.
armosec

Practical approaches

Use a shared capability schema so agents can describe skills consistently.
arxiv

Build runtime discovery that observes actual behavior, not just declared metadata.
armosec

Maintain a continuously updated inventory tied to business and technical context.
ardoq

Support ranking and composition so the system can choose not just one agent, but the right combination.
arxiv

Simple example

A customer-support workflow might discover one agent that classifies incoming tickets, another that retrieves policy docs, and a third that drafts replies. A capability-discovery layer can match each role to the best available agent and compose them into one pipeline.
arxiv
